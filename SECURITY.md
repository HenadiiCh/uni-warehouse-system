# Security Policy

## Supported Versions

As this is an educational project developed within a university course, active security support is provided only for the current stable release.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of our software seriously. If you have discovered a security vulnerability in **OptiStock**, please follow these steps:

1.  **Do not open a public issue immediately** if the vulnerability could be exploited to harm users.
2.  Please submit a report via the **"Issues"** tab on GitHub, labeling it with `security`, or contact the developer directly (if contact details are provided on the GitHub profile).
3.  In your report, please include:
    * A description of the vulnerability.
    * Steps to reproduce the issue.
    * The potential impact.

### What to Expect
* Since this is an open-source educational project, response times may vary.
* We will aim to acknowledge your report within 48-72 hours.
* Once a vulnerability is confirmed, we will release a patch as soon as possible.

Thank you for helping make **OptiStock** more secure!
